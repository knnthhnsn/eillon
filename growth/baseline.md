## Conversion paths

### Beles restock (primary)
`Search/social → /beles or /prickly-pear-parfum → #waitlist form → POST /api/waitlist`  
Trust block: `.shop__restock-trust` above form (one email, no charge, privacy — EXP-005)  
Events: `restock_form_started`, `restock_form_submitted`, `restock_form_error`  
Product slug: `beles`

### The Letter (newsletter)
`Footer / about / journal / store → form (product_slug=all, source=newsletter|footer) → /api/waitlist`  
Success copy in `script.js` (`waitlistMessages.all`)

### Private Copenhagen appointment
`mailto:care@eillon.maison` with subject variants  
Event: `studio_appointment_click` (homepage footer, nav footer)  
No booking form — email handoff

### Chapter waitlists (secondary)
`/asmara`, `/massawa`, `/ritual` → `#waitlist` → API

### Dynamic store cards
`script.js` renders mini-waitlist on eligible product cards

## Technical risks

| Risk | Severity | Notes |
|---|---|---|
| Static site — no SSR | low | Acceptable for brand site |
| Lighthouse perf on homepage | medium | Hero media + GSAP pins; ongoing optimization |
| Vercel deploy quota | medium | May block prod deploys; homepage `home.js` temporarily on jsDelivr CDN |
| Waitlist admin exposed path | low | Protected by API key + robots |
| No lint/typecheck | low | CI uses verify scripts + build |
| Analytics prod-only | medium | Local/staging hard to validate events |

## Obvious bottlenecks

1. **Beles is only purchasable path** — everything should funnel to restock or Letter
2. **Journal → Beles internal links** underdeveloped (EXP-008 next)
3. **Appointment path** is mailto-only (fine, but under-tracked on about page — EXP-027)
4. **No campaign content library** in repo for social loops (EXP-034)
5. **Measurement gaps** on newsletter-specific events (EXP-007)
6. **High-intent SEO pages still missing** for skin scent / oil-rich parfum clusters (EXP-013, EXP-023)
